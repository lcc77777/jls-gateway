# JLS 工业化套件：部署与操作说明 (v1.1)

本文件夹 (`jls_factory/`) 包含了将 JLS 从“原型交互”转为“工业化生产”的所有核心资产。

---

## 一、 文件定义 (这是什么？)

### 1. `model_list.json` (核心配置文件)
*   **用途**：JLS 的“产品百科”。定义了所有卡带型号（M5W, VNM 等）、对应的分类颜色、名字，以及最重要的 **LISP 固件代码**。
*   **重要性**：**修改这一个文件就够了。** 它是数据生成的唯一来源。

### 2. `generate_codes.py` (自动化引擎)
*   **用途**：读取 `model_list.json`，自动执行两项工作：
    1.  **编解码**：把 LISP 里的复杂符号转为安全的 Base64 字符串（存入数据库）。
    2.  **批量生产**：为每个型号生成数千个 12 位激活码（如 `JLS-K01-ABCD-1234`）。
*   **输出**：会生成两个 CSV 文件和一批 QR 二维码。

### 3. `supabase_schema.sql` (数据库蓝图)
*   **用途**：在 Supabase 里“盖房子”。
    *   创建 `models_registry`：存放卡带的“母本”信息（固件、颜色）。
    *   升级 `logic_inventory`：建立与母本表的关联。
    *   创建 `transactions`：记录激活后的对账单（送积分的凭证）。

### 4. `models_import.csv` & `inventory_import.csv` (装箱数据)
*   **用途**：运行 Python 脚本后的产物，用于导入 Supabase。它们是正式的“生产数据”。

---

## 二、 Supabase 从零部署步骤 (怎么操作？)

> [!WARNING]
> **之所以你现在看不到 `models` 表，是因为之前运行 SQL 时报错了（外键冲突）。请务必执行以下“清空再运行”步骤。**

### 第一步：清理老环境
1.  进入 Supabase 的 **Table Editor** -> `logic_inventory`。
2.  勾选所有现有的测试数据（比如你之前乱填的 1111...），点击 **Delete**。
    *   *道理：旧的测试码不符合现在的“工业标准”，会导致新旧表对不上。*

### 第二步：导入建筑蓝图 (SQL)
1.  进入 **SQL Editor**，打开你之前粘贴代码的那个选项卡。
2.  再次点击右下角的 **Run**。
3.  **确认看到绿色提示：** `Success. No rows returned.`
    *   *此时，左侧 Table Editor 里应该会出现 `models_registry` 和 `transactions` 两张新表。*

### 第三步：填入生产数据 (CSV)
1.  **先导入固件库**：Table Editor -> `models_registry` -> 点击 **Import** -> 拖入 `models_import.csv`。
2.  **后导入激活码**：Table Editor -> `logic_inventory` -> 点击 **Import** -> 拖入 `inventory_import.csv`。

## 四、 标准 SOP：如何上一个新卡带？ (New Model Onboarding)

当你和 Leo 开发了新的 LISP 固件（比如 V-Series 毒液系列），按以下 4 步操作即可：

### 1. 修改母本 (`model_list.json`)
打开该文件，复制一个现有的对象块，修改以下参数：
- `model_id`: 全球唯一标识，如 `JLS-V01-VENOM-V1`。
- `short_code`: 激活码前缀，如 `JLS-V01`。
- `name`: 中文名称。
- `lisp_code`: 直接把 Leo 写的 LISP 代码粘贴进去（**不需要你自己转码，脚本会处理**）。
- `quantity`: 这一批你想印多少张。

### 2. 运行生产引擎
在终端执行：
```bash
python3 jls_factory/generate_codes.py
```
*   **结果**：两个 CSV 文件会被全量更新，`qr_codes/` 文件夹下会多出新的一批二维码图片。

### 3. 上传固件母本 (Supabase)
1. 进入 Table Editor -> `models_registry`。
2. 点击 **Import** -> 拖入 `models_import.csv`。
3. **关键点**：Supabase 会自动识别由于 `model_id` 相同而导致的冲突。你可以选择 **"Upsert"** (更新已有的，插入没有的) 或直接点击导入。

### 4. 导入激活码 (Supabase)
1. 进入 Table Editor -> `logic_inventory`。
2. 点击 **Import** -> 拖入 `inventory_import.csv`。
3. 系统会自动把新生成的激活码补充进去。

## 五、 数据管理核心逻辑

*   **增量还是全量？** 
    Supabase 的导入功能非常智能。当你第二次导入 `inventory_import.csv` 时，它会发现之前的 5000 个码已经存在了，它只会提示报错或者跳过，**只有新生成的码会被存进去**。
*   **如何修改现有代码？**
    如果你要修复某个卡带的 LISP 漏洞，只需在 `model_list.json` 里改完代码，重新运行脚本，然后在 `models_registry` 表重新 Import 一次，选择 **"Upsert"** 模式覆盖即可。

## 🛠️ 环境运行提示
如果你在运行 Python 脚本或 `bridge.py` 时遇到权限或库错误：
1. **优先使用**: `./.venv/bin/python bridge.py ...`
2. **生成代码**: `python3 jls_factory/generate_codes.py` (确保已 `pip install qrcode`)
