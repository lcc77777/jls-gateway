# Rive 动画 Node.js 自动化与预览指南

本教程旨在帮助开发者在本地 Node.js 环境中实现 Rive (.riv) 动画的自动化读取、状态操作及浏览器预览。

## 1. 环境准备

首先安装必要的核心依赖。`@rive-app/canvas-advanced` 用于处理 Rive 逻辑，`canvas` 用于在 Node 环境下模拟图形渲染。

```bash
npm install @rive-app/canvas-advanced canvas
```

## 2. 核心踩坑点及修复方案

在 Node.js 中运行 Rive 存在以下技术障碍，本方案已全部解决：

- **WASM 加载失败**：Node 无法像浏览器一样自动定位 `.wasm`。需手动解析路径并注入 `wasmBinary`。
- **缺失浏览器环境**：Rive 运行时需要 `document` 和 `navigator`。脚本通过 Polyfill 进行了仿真。
- **CORS 跨域拦截**：浏览器出于安全考虑，禁止通过 `file://` 协议加载 `.riv` 二进制文件。必须使用 HTTP 服务器（如 `npx serve`）中转。

## 3. 自动化分析脚本 (`scripts/rive-auto.js`)

该脚本用于在命令行分析动画结构，获取 Artboard 和 State Machine 信息。

```javascript
const rive = require('@rive-app/canvas-advanced');
const fs = require('fs');
const path = require('path');
const { createCanvas } = require('canvas');

// 关键修复：环境仿真 (Polyfill)
global.document = {
    createElement: (type) => {
        if (type === 'canvas') return createCanvas(100, 100);
        return {};
    }
};
global.window = {};
global.navigator = { userAgent: 'node' };

async function runAutomation(filePath) {
    const rivBuffer = fs.readFileSync(filePath);
    const Runtime = rive.default || rive;

    // 关键修复：手动定位并加载 WASM
    const wasmPath = path.join(__dirname, '../node_modules/@rive-app/canvas-advanced/rive.wasm');
    const wasmBuffer = fs.readFileSync(wasmPath);

    const Rive = await Runtime({ wasmBinary: wasmBuffer });
    const file = await Rive.load(new Uint8Array(rivBuffer));
    const artboard = file.defaultArtboard();

    console.log(`Successfully loaded Artboard: ${artboard.name}`);

    // 遍历并分析状态机
    for (let i = 0; i < artboard.stateMachineCount; i++) {
        const sm = artboard.stateMachineByIndex(i);
        console.log(`State Machine [${i}]: ${sm.name}`);
    }

    artboard.delete();
    file.delete();
}

const targetFile = process.argv[2] || 'animation.riv';
runAutomation(targetFile).catch(console.error);
```

## 4. 浏览器预览生成 (`scripts/generate-html.js`)

生成一个 HTML 容器，通过 CDN 加载运行时来渲染本地的 `.riv` 文件。

```javascript
const fs = require('fs');

function generatePreview(rivFile) {
    const html = `
<!DOCTYPE html>
<html>
<body>
    <canvas id="canvas" style="width:100vw; height:100vh;"></canvas>
    <script src="https://unpkg.com/@rive-app/canvas@latest"></script>
    <script>
        const r = new rive.Rive({
            src: "${rivFile}",
            canvas: document.getElementById("canvas"),
            autoplay: true,
            onLoad: () => r.resizeDrawingSurfaceToCanvas()
        });
    </script>
</body>
</html>`;
    fs.writeFileSync('preview.html', html);
    console.log('preview.html generated.');
}

generatePreview(process.argv[2] || 'animation.riv');
```

## 5. 标准操作流程

1. **放置素材**：将你的 `.riv` 文件放在根目录（建议重命名为 `animation.riv`）。
2. **本地校验**：
   ```bash
   node scripts/rive-auto.js animation.riv
   ```
3. **网页输出**：
   ```bash
   node scripts/generate-html.js animation.riv
   ```
4. **启动预览**（解决 CORS 问题）：
   ```bash
   npx serve .
   ```
   然后在浏览器打开地址（通常是 `http://localhost:3000/preview.html`）。
